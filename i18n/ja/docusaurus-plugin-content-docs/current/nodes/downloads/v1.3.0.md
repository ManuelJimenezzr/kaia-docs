## Klaytn v1.3.0のリリースノート

## お知らせ<a id="notice"></a>

- klaytn v1.3.0の公式goバージョンは、セキュリティ問題に対処するため、go-1.13.3となりました。 klaytnのビルドにはgo-1.13.3以上のバージョンを推奨します。

### 新機能<a id="new-features"></a>

- RPC API (klay_signTransaction および klay_signTransactionAsFeePayer) による Klaytn トランザクション・タイプの署名。
- TxTypeFeeDelegatedChainDataAnchoring および TxTypeFeeDelegatedChainDataAnchoringWithRatio が追加されました。 これで、サービスチェーンからのアンカリング取引の取引手数料は、別の手数料支払口座から支払うことができる。
- 新しいJSON-RPC API
  - klay.encodeAccountKey
  - klay.decodeAccountKey
  - klay.sendTransactionAsFeePayer
  - subbridge.setParentOperatorFeePayer
  - サブブリッジ.parentOperatorFeePayer
  - subbridge.setChildOperatorFeePayer
  - サブブリッジ.childOperatorFeePayer

### 修正<a id='fixes'></a>

- 状態更新スキップによる間違った不良ブロックの発生を修正。
- call/estimateGas/estimateComputationCostを実行した際に、送信者の誤った残高を返すことがあったのを修正した。
- タイムアウト前にキューに入れられたトランザクションが失われる問題を修正。

### 改善点<a id='improvements'></a>

- ラウンドチェンジのタイマーを3秒に変更し、より早いリカバリーを実現
- CN間の最適化されたコンセンサスメッセージ伝播
- 安定性向上のためのテスト・コードの追加
- パッケージの説明を追加
- 洗練されたパッケージング・スクリプト

## リムーバル<a id='removals'></a>

- 未使用のパッケージ 'filter' を削除。
- 未使用のJSON-RPC APIを削除
  - miner "名前空間のAPI
  - klay.mining
  - klay.submitワーク
  - klay.getWork
  - klay.submitハッシュレート
