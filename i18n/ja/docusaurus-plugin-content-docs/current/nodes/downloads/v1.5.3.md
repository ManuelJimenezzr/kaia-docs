## リリースノート

### 新機能

- 取得したブロックチェーンデータをリアルタイムでKafkaにプッシュするストリーミングモジュールを実装。
  - Kafkaクライアントを実装しました。
  - Kafka用のオプションを追加しました。
  - Kafkaコンシューマを実装しました。
- リモートキャッシュ（redis）の実装 (#681)
- 最新ブロックにRedisのpub/subを実装した(#676)
- DBマイグレーション機能を実装 (#648)
- コールドスタート時のパフォーマンス向上のため、再起動後にキャッシュを復元するStateTrieNodeキャッシュダンプ機能を実装しました。

### 改善点

- 状態トライのマイグレーションにおいて、間違った状態API出力を修正 (#703)
- AccountKeyLegacy\`を含むロールベースのキーの検証ロジックを修正しました。

### その他

- Fetcher/Downloader/Workerを無効にするオプションを追加 (#680, #688)。
- 既存のチェーンデータでジェネシスブロックを生成するようにした(#649)
- 未使用のフラグ `--cache.writethrough` を削除しました。
- トライDBの読み込みに失敗した場合のエラーログを追加 (#683)
- ステーキング契約のソースコードを追加 (#699)
- 過去のブロックをテストするために `--start-block-num` を追加 (#711)
- キーストア作成時の有効性チェックを追加しました。
- レベルDBに読み書きのパフォーマンスメトリクスを追加しました。
