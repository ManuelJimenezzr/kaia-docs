## Klaytn v1.4.0のリリースノート

### 新機能<a id="new-features"></a>

- キーストアV4
  - Klaytnのアカウントをサポートするために、新しいキーストア標準[KIP-3](https://klaytn.github.io/kips/KIPs/kip-3)が採用された。

### 修正<a id='fixes'></a>

- `stakingInfo`更新時のタイミングの問題による断続的な不良ブロックの発生を修正。
- `debug.traceBlock`の実行中に、APIとトライノードのガベージコレクションのタイミングの問題でパニックが発生するのを修正した。
- ブロック同期中のバグ報告を修正。

### 改善点<a id='improvements'></a>

- 最適化されたストレージトライ
  - Klaytnはトランザクションごとではなく、ブロックごとにストレージルートのハッシュを計算するようになった。
  - Klaytnはトライのハッシュ根を同時に計算する。
- ステート・トライ・キャッシュ・サイズの増加
  - パフォーマンスを向上させるため、デフォルトのキャッシュサイズを大きくした。
- Badger DBのバージョンをv1.6.0にアップグレードしました。
- メトリクス・パッケージを交換
- パッケージの説明を追加
- テストコードカバレッジの向上
