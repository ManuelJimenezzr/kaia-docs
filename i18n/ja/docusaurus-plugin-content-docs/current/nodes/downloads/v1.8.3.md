## リリースノート

Klaytn v1.8.3では、ブロックの生成時やトランザクションの伝播時に、各ノードで受信した時刻に従ってトランザクションが時系列順にソートされる。 以前は、取引はガス価格でソートされていた。 しかし、クレイトンのガソリン価格は固定されているため、基本的に注文はなかった。 このバージョン変更に伴い、Klaytnノードは先着順でトランザクションを処理します。

### 新機能

- 同じガス価格の取引は、受付時間順にソートされます。 (#1282)
- また、伝播されるトランザクションは、受信した時間順にソートされる。 (#1309)
- ファイルに書き込まれるジャーナル・トランザクションは、受信時刻順にソートされる。 (#1329)

### 改善点

- プリフェッチの無効化オプションが利用可能になり、スナップショットに不足していたメトリクスが追加されました。 (#1234)
- GetCodeSizeはemptyCodeHashに対してエラーではなく0を返すようになりました。 (#1279)

### 修正

- Worker の署名者生成関数を `LatestSignerForChainID` から `MakeSigner` に変更し、`SenderCacher` を使えるようにした。 (#1305)
- プリフェッチャーのパニックガードを追加。 (#1311)

### その他

- 組み込みのソリディティ試験ファイルをv0.8.xにアップグレード。 (#1149 #1189)
- go1.18をサポートし、CircleCi用のDockerfileをAlpineからUbuntu 20.04に更新した。 (#1320)
- 軽整備（#1265 #1270 #1273 #1308 #1301 #1306 #1312 #1313 #1314 #1315 #1322 #1327 #1338 #1349）
