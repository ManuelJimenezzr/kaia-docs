## 发布说明

使用 Klaytn v1.8.3，在生成区块或传播事务时，事务会根据每个节点的接收时间按时间顺序排序。 之前，交易是按天然气价格排序的。 但由于 Klaytn 的天然气价格是固定的，他们基本上没有订单。 版本更改后，Klaytn 节点将以 "先到先得 "的方式处理交易。

### 新功能

- 气价相同的交易将按接收时间排序。 (#1282)
- 正在传播的交易也将按照接收时间的先后顺序排序。 (#1309)
- 写入文件的日记账事务将按接收时间顺序排序。 (#1329)

### 改进

- 现在可以禁用预取选项，并添加了快照的缺失指标。 (#1234)
- 对于 emptyCodeHash，GetCodeSize 现在将返回 0，而不是错误。 (#1279)

### 修复

- 将 Worker 的签名者生成函数从 "LatestSignerForChainID "改为 "MakeSigner"，以便它可以使用 "SenderCacher"。 (#1305)
- 为预取器添加了紧急保护。 (#1311)

### 杂项

- 将内置稳固性测试文件升级至 v0.8.x。 (#1149 #1189)
- 支持 go1.18，并将 CircleCi 的 Dockerfile 从 Alpine 更新到 Ubuntu 20.04。 (#1320)
- 小规模维修（#1265 #1270 #1273 #1308 #1301 #1306 #1312 #1313 #1314 #1315 #1322 #1327 #1338 #1349）
